Depth AI's Bitbucket integration enables AI-powered code understanding, documentation assistance, and intelligent code search across your Bitbucket repositories. This guide provides detailed steps to set up and configure the integration.

<Steps>
  <Step title="Access Integration Settings">
    1. Log into your Depth AI dashboard
    2. Navigate to [Integration Settings](https://app.trydepth.ai/new-settings/integrations)
    3. Locate the Bitbucket section
    4. Click "Connect" to initiate the integration
    
    <Frame>
      <img src="/images/bitbucket/connect.png" alt="Bitbucket Connect Button" />
    </Frame>
  </Step>

  <Step title="Authorize Depth AI">
    You'll be redirected to Bitbucket's authorization page where you'll need to:
    1. Review the requested permissions
    2. Select your workspace (if you have multiple)
    3. Click "Grant access"
    
    <Frame>
      <img src="/images/bitbucket/authorize.png" alt="Bitbucket Authorization Screen" />
    </Frame>

    <Info>
      For organization accounts, you may need administrator approval to install the integration. Contact your Bitbucket workspace admin if you don't have sufficient permissions.
    </Info>
  </Step>

  <Step title="Configure Repository Access">
    After authorization, you'll be prompted to configure repository access:

    1. Choose access level:
       - **All repositories** - Recommended for team-wide usage
       - **Selected repositories** - Better for specific project focus
    
    2. Set repository permissions:
       - Code reading
       - Pull request access
       - Repository metadata
    
    <Frame>
      <img src="/images/bitbucket/repo-select.png" alt="Repository Selection Screen" />
    </Frame>

    <Warning>
      Only grant access to repositories that will actively use Depth AI features. You can modify these permissions later in your Bitbucket workspace settings.
    </Warning>
  </Step>
</Steps>

## Detailed Permissions Overview

Depth AI requires specific permissions to provide its full feature set:

### Repository Code Access
- **Read access to source code**: Enables AI to understand your codebase
- **Branch information**: Allows tracking of different code versions
- **Commit history**: Provides context for code changes

### Pull Request Integration
- **PR content reading**: Enables code review assistance
- **PR metadata access**: Allows tracking PR status and updates
- **Comment access**: Enables AI to understand discussion context

### Repository Metadata
- **Repository settings**: Basic repository configuration access
- **User permissions**: Understanding of team structure
- **Project organization**: Access to project structure and organization

<Info>
These permissions are essential for features like:
- Intelligent code search across repositories
- Context-aware code explanations
- Automated documentation assistance
- Code review suggestions
</Info>

## Advanced Configuration

### Workspace Settings
Fine-tune your integration by configuring:
- Repository-specific access levels
- Branch protection rules compatibility
- Custom webhook settings (if needed)

### Security Considerations
- Regular audit of connected repositories
- Periodic review of granted permissions
- IP allowlist configuration (for enterprise setups)

## Troubleshooting Guide

### Common Issues and Solutions

1. **Connection Failures**
   - Verify Bitbucket account permissions
   - Check workspace access settings
   - Ensure no conflicting integrations

2. **Repository Access Issues**
   - Confirm repository visibility settings
   - Check workspace permission levels
   - Verify branch protection rules

3. **Authorization Errors**
   - Clear browser cache and cookies
   - Re-authenticate Bitbucket account
   - Check for expired tokens

4. **Integration Not Working**
   - Verify webhook configurations
   - Check repository access permissions
   - Ensure repository is properly indexed

<Note>
If issues persist, check our [status page](https://status.trydepth.ai) or contact support with your Bitbucket workspace ID and affected repository names.
</Note>
